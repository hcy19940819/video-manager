# VideoFaceManager 视频人物花名册系统

**从已分类视频自动学习人物特征，然后识别新视频**

---

## 工作流程

```
已分类视频/                    ← 你提供的
├── 张三/                      ← 系统自动学习"通用脸"
│   ├── video1.mp4
│   └── video2.mp4
├── 李四/
└── 王五/

        ↓ 运行 learn 命令
        ↓ 系统自动提取每个人物的"标准脸"

未分类视频/                    ← 自动识别
├── unknown1.mp4     →  识别出: 张三 (85%)
├── unknown2.mp4     →  识别出: 李四+王五 (72%)
└── ...

        ↓ 运行 identify 命令
        ↓ 生成标识文件

已识别/
├── 张三/
│   └── [视频链接]
├── 李四/
│   └── [视频链接]
├── 王五/
│   └── [视频链接]
├── 未识别/
│   └── [视频链接]
├── 识别报告.json              ← 详细数据
└── 人物标签.txt               ← 人读格式
```

## 快速开始

### 第一步：准备已分类视频

把你已经分好类的视频放到一个文件夹里：

```
D:/已分类视频/
├── 张三/
│   ├── 聚会视频1.mp4
│   ├── 生日视频.mp4
│   └── 旅游视频.mp4
├── 李四/
│   ├── 工作记录.mp4
│   └── 会议视频.mp4
└── 王五/
    └── 采访视频.mp4
```

**要求**：
- 每个人物一个文件夹
- 文件夹名字就是人物名
- 每个文件夹至少3-5个视频（更多更好）

### 第二步：学习人物特征

```bash
python cli.py learn "D:/已分类视频"
```

系统会：
1. 扫描每个人物文件夹
2. 从视频中提取人脸
3. 生成"通用脸"特征
4. 保存人脸截图到 `data/faces/`

输出示例：

### 第三步：识别新视频

```bash
python cli.py identify "D:/未分类视频" -o "D:/已识别" --auto-move
```

参数说明：
- `"D:/未分类视频"` - 待识别的视频文件夹
- `-o "D:/已识别"` - 输出目录
- `--auto-move` - 自动按人物分类（创建软链接/复制）
- `-c 0.7` - 设置置信度阈值（默认0.65）

## 提高识别准确度

### 1. 学习阶段
- **视频越多越好**：每人至少5个视频，20个以上最佳
- **场景多样化**：不同光线、角度、表情
- **视频要清晰**：模糊视频会提取错误特征

### 2. 识别阶段
- **调整置信度**：`-c 0.7` 提高门槛，`-c 0.5` 放宽
- **二次筛选**：检查"未识别"文件夹，手动补充

### 3. 迭代改进
如果识别效果不好：
```bash
# 1. 把错误分类的视频移到正确文件夹
# 2. 重新学习
python cli.py learn "D:/已分类视频"

# 3. 重新识别
python cli.py identify "D:/未分类视频" -o "D:/已识别" --auto-move
```

## 常见问题

### Q: 识别太慢怎么办？
- 降低采样频率：修改 `core/config.py` 中的 `SAMPLE_INTERVAL`
- 缩短视频时长：先用剪辑软件把视频剪短
- 使用CPU模式：默认就是用CPU，AMD显卡不支持CUDA加速

### Q: 识别不准确怎么办？
- 增加学习样本：每人提供更多视频
- 调整置信度：`--min-confidence 0.8` 只保留高置信度结果
- 手动纠正：在Web面板中编辑人物标签

### Q: 多个人物同框怎么识别？
- 系统会识别画面中最大的人脸
- 如果多人都重要，会分别识别并列出

### Q: 可以识别人群中的次要人物吗？
- 默认只识别最大人脸（主角）
- 如需识别所有人，需要修改代码逻辑
