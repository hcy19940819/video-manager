# 视频管家 - 一体化视频管理系统

**一次扫描，数据互通**

整合了视频去重 + 人物识别 + 以图搜视频 + **库管理可视化**功能。

---

## 🆕 新增功能：库管理可视化（重点！）

全新的 **Web 可视化界面**，类似影库管家的库标记功能：

- ✅ **添加库** - 添加视频文件夹，自动检测分类状态
- ✅ **标记状态** - 标记库为：未分类 / 分类中 / 已分类
- ✅ **关联人物** - 标记库对应的人物
- ✅ **智能检测** - 自动分析文件夹结构，判断分类方式
- ✅ **可视化面板** - 美观的 Web 界面，一览所有库的状态

```bash
cd videomanager

```

![功能概览](https://via.placeholder.com/800x400?text=Web+界面预览)

**界面功能：**
- 📊 **统计面板** - 显示未分类/分类中/已分类的库数量

## 智能分类检测

系统会自动分析文件夹结构，判断分类方式：

| 检测类型 | 说明 | 示例 |
|---------|------|------|
| **按人分类** | 子文件夹名是人名 | `张三/`, `李四/` |
| **按类型分类** | 子文件夹是类型名 | `电影/`, `电视剧/` |
| **按日期分类** | 子文件夹是日期 | `2024-01/`, `2023年/` |
| **扁平结构** | 没有子文件夹 | 所有视频在根目录 |
| **混合/未分类** | 结构不清晰 | 需要人工整理 |

**检测结果会显示：**
- 分类类型（按人/按类型/按日期/混合）
- 置信度（0-100%）
- 视频分布（每个文件夹的视频数量）
- 建议操作

## 完整工作流

### 场景1：已有分类好的视频

```
视频/
├── 张三/
│   ├── 视频1.mp4
│   └── 视频2.mp4
└── 李四/
    └── 视频3.mp4
```

操作步骤：
1. 启动 Web 界面：`./启动界面.sh`
2. 点击「添加库」，输入路径 `/视频`
3. 系统自动检测为「按人分类」，置信度 90%+
4. 状态自动标记为「已分类」
5. 关联人物自动填入：张三、李四
6. 使用「学习」功能让人脸识别学习这些人物

### 场景2：大量未分类视频需要整理

操作步骤：
1. 启动 Web 界面
2. 添加库，标记状态为「未分类」
3. 扫描视频生成指纹和人脸数据
4. 通过「找重复」清理重复文件
5. 通过「人物库」查看识别出的人物
6. 逐步整理，将状态改为「分类中」→「已分类」

## 核心功能：视频去重 + 人物识别

### 扫描视频

```bash
# 快速模式（默认）- 去重 + 适中AI采样
python 视频管家.py 扫描 "/视频文件夹"

# 简单模式 - 仅去重，不做AI识别（最快）
python 视频管家.py 扫描 "/视频文件夹" --模式 简单

# 深度模式 - 去重 + 密集AI采样（最准但慢）
python 视频管家.py 扫描 "/视频文件夹" --模式 深度

# 以后新增视频，只扫新增的
python 视频管家.py 扫描 "/视频文件夹" --增量

# 硬盘保护（每100GB休息10秒）
python 视频管家.py 扫描 "/视频文件夹" --硬盘保护 100

# 关闭硬盘保护
python 视频管家.py 扫描 "/视频文件夹" --硬盘保护 0
```

#### 扫描模式对比

| 模式 | 用途 | AI识别 | 采样间隔 | 速度 | 适用场景 |
|------|------|--------|----------|------|----------|
| **简单** | 仅去重 | ❌ 不做 | - | ⚡⚡⚡ 最快 | 大量视频初次去重 |
| **快速** (默认) | 去重+AI | ✅ 适中 | 10秒/帧 | ⚡⚡ 较快 | 日常使用推荐 |
| **深度** | 深度AI | ✅ 密集 | 3秒/帧 | ⚡ 较慢 | 精确人脸识别 |

#### 硬盘保护机制

```bash
# 默认每100GB休息10秒
python 视频管家.py 扫描 "/视频文件夹"

# 自定义批量大小（50GB一休息）
python 视频管家.py 扫描 "/视频文件夹" --硬盘保护 50

# 对固态硬盘可以关闭保护
python 视频管家.py 扫描 "/视频文件夹" --硬盘保护 0
```

## 数据库说明

### 去重库 (去重库.db)
存储视频指纹，用于找重复：
- 视频路径
- 文件大小
- 快速哈希（aHash）
- 文件哈希（MD5前1MB）
- 时长、分辨率

### 人物库 (人物库.db)
存储人物识别结果：
- 视频路径
- 人物名
- 出现时间点
- 置信度
- 人脸截图路径

### 库管理库 (库管理.db)
存储库的标记信息：
- 库路径
- 分类状态（未分类/分类中/已分类）
- 关联人物
- 视频数量、总大小、总时长
- 检测备注

## 注意事项

1. **Web 界面默认端口 5000**，如果被占用可以修改 `库管理界面.py` 里的 `port=5000`
2. **第一次扫描较慢** - 需要解码视频、计算哈希、识别人脸
3. **增量扫描很快** - 只处理新增的文件
4. **人脸识别的准确性** - 依赖学习数据，已分类视频越多越准
5. **库管理只记录元数据** - 删除库不会删除实际视频文件
6. **硬盘保护建议** - 机械硬盘建议保持默认(100GB)，固态硬盘可关闭
7. **扫描模式选择** - 
   - 初次大批量去重用 `简单` 模式
   - 日常使用用 `快速` 模式（默认）
   - 需要精确识别用 `深度` 模式

## 问题排查

**提示"需要安装 flask"**
```bash
pip install flask
```

**提示"端口被占用"**
```bash
# 修改库管理界面.py 里的端口
app.run(host='0.0.0.0', port=5001)  # 改成其他端口
```

**扫描时硬盘噪音大/发热严重**
```bash
# 减小批量大小，增加休息频率
python 视频管家.py 扫描 "/视频文件夹" --硬盘保护 50  # 每50GB休息

# 或使用简单模式减少磁盘读取
python 视频管家.py 扫描 "/视频文件夹" --模式 simple
```

**扫描速度太慢**
```bash
# 使用简单模式（不做AI识别）
python 视频管家.py 扫描 "/视频文件夹" --模式 简单

# 或关闭硬盘保护（固态硬盘适用）
python 视频管家.py 扫描 "/视频文件夹" --硬盘保护 0
```

**其他问题**
- 需要安装 opencv-python: `pip install opencv-python pillow numpy`
- 人脸识别模型未加载: `pip install insightface`
- 扫描中断后数据丢失：每10个视频会自动保存，重新扫描会从断点继续
