## 需求总览

### 核心需求
1. **自动转存**: 从STboy论坛提取的网盘链接自动转存到个人网盘
2. **断点续传**: 任务中断后可从断点继续，不重复处理
3. **多网盘支持**: UC网盘、百度网盘、迅雷网盘、夸克网盘
4. **进度显示**: 黑窗实时显示进度条和处理状态

6. **异常记录**: 记录提取码错误、验证码等异常情况

### 性能需求
- 迅雷网盘: 7窗口并行，动态补满，刷新重试
- 其他网盘: 单窗口串行处理

## 功能实现详情

### 1. 统一入口 (网盘转存工具.py)

**功能**:
- 数字菜单选择网盘 (1-4)
- 显示校准状态 (已校准/未校准/断点续传)
- 校准入口 (10)
- 通用坐标读取 (10.10)

**菜单**:
```
1. UC网盘 (已校准)
2. 百度网盘 (断点续传 50/100)
3. 迅雷网盘 (已校准)
4. 夸克网盘 (未校准)
10. 校准坐标
0. 退出
```

---

### 2. 断点续传 (progress.py)

**实现**:
- 进度文件: `progress/{pan_type}_progress.json`
- 记录: folder, completed[], failed[], total, last_update
- 自动检测未完成进度
- 支持继续/取消/重新下载

**接口**:
- `load_progress(pan_type)` - 加载进度
- `save_progress(...)` - 保存进度
- `clear_progress(pan_type)` - 清除进度
- `get_pending_links(...)` - 获取待处理链接

---

### 3. 窗口状态检测 (window_inspector.py)

**实现**:
- 使用 `win32gui.EnumChildWindows` 遍历控件
- `GetWindowText` 读取控件文字
- 关键词匹配识别状态

**支持状态**:
| 状态 | 百度 | UC | 迅雷 | 夸克 |
|------|------|-----|------|------|
| 提取码错误 | ✅ | ✅ | ✅ | ✅ |
| 链接失效 | ✅ | ✅ | ✅ | ✅ |
| 文件被删 | ✅ | ✅ | ✅ | ✅ |
| 需要登录 | ✅ | ✅ | ✅ | ✅ |
| 验证码 | ✅ | ❌ | ✅ | ❌ |
| 转存成功 | ✅ | ✅ | ✅ | ✅ |
| 下载限制 | ✅ | ✅ | ✅ | ✅ |

---

### 4. 迅雷网盘 (xl.py) - 最复杂

**核心特性**:
- **7窗口并行**: MAX_WINDOWS = 7
- **动态补满**: 窗口完成立即开新链接
- **刷新重试**: 卡住时刷新重试(最多2次)
- **智能检测**: 实时检测每个窗口状态

**坐标配置**:
```python
TAB_COORDS = {'x': 765, 'y': 333}      # 第1个标签页
TAB_WIDTH = 100                         # 标签页宽度
REFRESH_COORDS = {'x': 750, 'y': 366}  # 刷新按钮
```

**窗口状态机**:
- `pending` - 待处理
- `processing` - 处理中(点击转存后)
- `success` - 成功
- `failed` - 失败
- `stuck` - 卡住(超时)
- `closed` - 已关闭

**流程**:
1. 初始化打开7个链接
2. 等待5秒加载
3. 循环处理:
   - 遍历7个窗口检测状态
   - success/failed → 关闭 → 开新链接补满
   - stuck → 刷新重试
   - processing → 继续检测
4. 全部完成退出

**日志**: `logs/xunlei_文件夹名_时间.log`  
**异常**: `异常文件夹/文件夹名/xunlei_类型.txt`

**流程**:

2. 检测页面状态(5秒超时)
3. 点击转存
4. 检测转存结果(8秒超时)
5. 点击关闭

**日志**: `logs/baidu_文件夹名_时间.log`  
**异常**: `异常文件夹/文件夹名/baidu_类型.txt`
