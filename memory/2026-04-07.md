### 代码审查：迅雷网盘自动转存工具

**问题分析** (xl.py + window_inspector.py)

1. **核心Bug**: `detect_xunlei_status()` 读取到错误标签页的状态
   - 原因：`EnumChildWindows` 遍历控件时，迅雷隐藏标签页的内容控件仍返回 `IsWindowVisible = True`
   - 导致：同时开了7个标签时，读到了"密码错误"但实际显示的是"保存成功"

2. **迅雷特性**: 关闭标签后，后面的标签自动前移补位
   - 始终处理索引0即可，队列自然推进
   - 但这也暴露了检测读错来源——隐藏标签的内容区仍"可见"

**优化方案讨论** (保留7标签并行)

| 方案 | 原理 | 实现难度 |
|------|------|---------|
| 区域过滤 | 只读取内容区(y>100, y<底部50)的文字 | 中等 |
| Z序过滤 | 只取最前面5个控件的文字 | 简单 |
| 权重法 | 位置越靠前的文字权重越高 | 简单 |
| 先聚焦再检测 | 点击标签→等0.3s→检测，确保是当前页 | 中等 |

**推荐组合**: 点击聚焦 + 区域过滤 + 只读前N个控件

### 项目：VideoDedup 视频快速去重工具

**背景**: 用户需要参考 DoubleKiller 的视频去重工具，但 DoubleKiller 只能识别完全相同的文件（MD5哈希），无法识别相同内容但不同格式/编码的视频。

**核心需求**:
1. 专门针对视频（而非通用文件）
2. 支持增量对比（保存指纹数据库，只处理新文件）
3. 保存对比路径（支持多库对比）
4. 识别不同格式/编码/轻微剪辑的相同内容视频

**技术方案**:
- **感知哈希 (pHash)**: 使用 DCT 变换生成 64 位哈希，对格式转换和压缩具有鲁棒性
- **多层指纹架构**: 
  - L1: 文件元数据（大小、时长、分辨率）→ 快速预筛选
  - L2: 快速 pHash（均匀采样10帧）→ 粗筛相似视频
  - L3: 详细 pHash（关键帧序列）→ 精确比对内容
  - L4: 文件 MD5 → 确认完全重复
- **SQLite 数据库存储**: 支持增量扫描，通过 mtime 判断文件变更

**已完成功能**:
1. ✅ CLI 命令行工具 (`videodedup.py`)
   - `scan` - 扫描视频生成指纹
   - `find` - 查找重复视频
   - `compare` - 跨库对比
   - `rename` - 重命名标记重复文件
   - `stats` - 显示统计信息

2. ✅ 重命名标记功能 (`rename` 子命令)
   - 添加 `_dup{id:03d}` 后缀（如 `_dup001`）
   - 同一组重复文件使用相同ID，方便手动整理
   - 支持预览模式和实际执行
   - 可导出重命名脚本（.sh/.bat）

3. ✅ 可视化HTML报告 (`html_report.py`)
   - 自动生成 `report.html` 可视化报告
   - 缩略图预览（视频中间帧截图）
   - 对比表格（大小、时长、分辨率并排显示）
   - 勾选删除功能
   - 搜索筛选
   - 导出删除/重命名脚本

4. ✅ GUI界面 (`videodedup_gui.py`)
   - 图形化操作界面
   - 支持扫描、查找、导出功能
   - 集成重命名功能

**使用流程** (推荐):
```bash
# 1. 扫描视频库
python videodedup.py scan ~/Videos

# 2. 查找重复（生成 .md, .json, .html）
python videodedup.py find

# 3. 打开可视化报告
open report.html

# 4. 在浏览器中勾选删除/导出脚本
```

**相似度阈值**:
- 汉明距离 ≤5: 视为相同
- 汉明距离 5-10: 视为相似
- 序列相似度 ≥0.9: 视为相同内容

**依赖项**: 
- 核心功能无需额外依赖
- HTML报告需要: `pip install opencv-python pillow numpy`
- 可选进度条: `pip install tqdm`
