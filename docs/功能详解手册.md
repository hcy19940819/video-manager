## 四层去重系统

### 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    四层去重检测流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第零层：预筛选                                               │
│  ├─ 条件：文件大小 + 视频时长                                  │
│  └─ 作用：快速排除明显不同的视频                               │
│         ↓                                                   │
│  第一层：文件哈希 (MD5)                                        │
│  ├─ 条件：文件内容完全一致                                     │
│  └─ 结果：identical (完全重复)                                │
│         ↓ 不匹配                                              │
│  第二层：快速pHash                                            │
│  ├─ 条件：视频中间帧感知哈希                                   │
│  ├─ 阈值：汉明距离 ≤ 5                                        │
│  └─ 结果：similar (相似视频)                                  │
│         ↓ 不匹配                                              │
│  第三层：详细pHash                                            │
│  ├─ 条件：多关键帧哈希序列对比                                  │
│  ├─ 阈值：50%以上关键帧匹配                                    │
│  └─ 结果：maybe (可能相似)                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 各层详解

#### 第零层：元数据预筛选

| 项目 | 说明 |
|------|------|
| **原理** | 比较文件大小和视频时长 |
| **作用** | 快速排除明显不同的视频，减少后续计算 |
| **逻辑** | 大小和时长都相同的才进入下一层检测 |

```
示例：
视频A: 100MB, 2:30
视频B: 100MB, 2:30  → 进入检测
视频C: 50MB,  1:45  → 直接排除
```

---

#### 第一层：文件哈希 (MD5)

| 项目 | 说明 |
|------|------|
| **原理** | 计算整个文件的MD5哈希值 |
| **精度** | 100% - 文件内容一字节不差 |
| **速度** | 慢（需要读取整个文件） |
| **结果** | **identical** (完全重复) |
| **适用场景** | 同一个视频被复制多份 |

```
检测结果示例：
┌─────────────────────────────────────────┐
│ ⚠️ 完全重复  相似度: 100%               │
│                                         │
│  • video_2024.mp4  (100MB)              │
│  • video_2024_copy.mp4  (100MB)         │
│                                         │
│  [对比预览] [删除选中] [保留首个]        │
└─────────────────────────────────────────┘
```

---

#### 第二层：快速pHash

| 项目 | 说明 |
|------|------|
| **原理** | 提取视频**中间帧**，计算感知哈希 |
| **算法** | DCT变换 → 8x8低频 → 汉明距离比较 |
| **阈值** | 汉明距离 ≤ 5 认为相似 |
| **速度** | 快（只读1帧） |
| **结果** | **similar** (相似视频) |
| **适用场景** | 同一视频不同压制版本 |

```
技术细节：
┌─────────────────┐
│ 视频文件         │
│   [=========]   │ ← 提取中间帧（50%位置）
└────────┬────────┘
         ↓
┌─────────────────┐
│ 中间帧图像       │
│  ┌─────────┐    │
│  │  画面   │    │ → DCT变换
│  └─────────┘    │
└────────┬────────┘
         ↓
┌─────────────────┐
│ pHash = 010110.. │ (64位二进制)
└─────────────────┘
```

**能检测的情况：**
- 不同码率的同一视频
- 不同分辨率（720p vs 1080p）
- 轻微压缩损失

**不能检测的情况：**
- 画面裁剪
- 开头/结尾不同（中间帧相同）

---

#### 第三层：详细pHash

| 项目 | 说明 |
|------|------|
| **原理** | 提取**多关键帧**，计算哈希序列 |
| **采样** | 5-20个关键帧（根据视频长度） |
| **阈值** | 50%以上关键帧匹配 |
| **速度** | 慢（需要解码多帧） |
| **结果** | **maybe** (可能相似) |
| **适用场景** | 编辑版本、剪辑片段 |

```
技术细节：
视频: [帧1][帧2][帧3]...[帧N]
       ↓    ↓    ↓      ↓
     采样  采样  采样   采样  ← 均匀间隔采样
       ↓    ↓    ↓      ↓
     Hash1 Hash2 Hash3 HashN
       ↓    ↓    ↓      ↓
     比较  比较  比较   比较  ← 逐对比较
       
匹配率 = 匹配数 / 总采样数 > 50% ? 相似 : 不同
```

**能检测的情况：**
- 剪辑版本（开头/结尾不同）
- 不同分辨率的同一内容
- 画面轻微变化

---

### 检测结果类型

| 类型 | 标识 | 说明 | 处理建议 |
|------|------|------|----------|
| **identical** | 🔴 完全重复 | 文件内容100%相同 | 放心删除重复份 |
| **similar** | 🟡 相似视频 | 内容相同，质量可能不同 | 保留最高质量版 |
| **maybe** | 🔵 可能相似 | 内容相似，可能有差异 | 人工对比后决定 |

## 花名册系统

### 概念

```
花名册 = 人物数据库

┌─────────────────────────────────────┐
│ 人物: 小宝                           │
├─────────────────────────────────────┤
│ 特征向量1  ← 从视频A提取的人脸        │
│ 特征向量2  ← 从视频B提取的人脸        │
│ 特征向量3  ← 从视频C提取的人脸        │
│ ...                                 │
├─────────────────────────────────────┤
│ 出现记录:                            │
│ • video_A.mp4 @ 00:30, 02:15        │
│ • video_B.mp4 @ 05:20               │
│ ...                                 │
└─────────────────────────────────────┘
```

---

### 使用流程

```
1. 准备训练素材
   └── 收集包含该人物的视频/图片
   
2. 学习人物
   👤 人物标签页
     └── 输入姓名 + 视频路径
     └── 点击 [开始学习]
     └── 📋 任务队列显示进度
     
3. 识别人物
   💬 "找包含小宝的视频"
   └── 系统自动搜索该人物出现的视频
```

---

### 训练建议

| 建议 | 说明 |
|------|------|
| 视频越多越好 | 不同角度、光线，提高识别率 |
| 正面清晰最佳 | 侧脸、遮挡会降低准确率 |
| 多视频混合 | 不要只用同一个视频 |
| 单人视频优先 | 多人视频也能学，但效率低 |

## 常见问题

**Q: 为什么有的重复没检测出来？**
A: 可能原因：
- 视频被大幅裁剪（用深度模式重扫）
- 分辨率差异太大
- 只扫了简单模式，未计算pHash

**Q: 人物识别不准怎么办？**
A: 建议：
- 用深度模式重新学习
- 提供更多训练视频
- 确保人脸清晰正面

**Q: 扫描太慢怎么办？**
A: 建议：
- 使用简单模式（跳过人脸识别）
- 分批扫描小文件夹
- 调整硬盘保护参数
