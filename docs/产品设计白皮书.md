# 🎬 视频管家 v4.0 - 产品设计白皮书

> **产品定位**：最懂你的AI视频管家 - 像助理一样对话，像网盘一样安全
> 
> **核心场景**：个人视频整理（家庭录像、下载的电影）
> **交互方式**：自然语言 + 可视化界面
> **文件处理**：非侵入式（不碰原文件）

## 🎯 核心功能流程

### 流程 1：初次使用

```
用户: "帮我整理视频"
    ↓
AI:  "好的！我发现了几个视频文件夹：
      1. /家庭录像 - 125个视频
      2. /Downloads - 15个视频
      3. /备份 - 300个视频
      
      建议先扫描 /家庭录像，这是您的主要视频库吗？"
    ↓
用户: "是的，先扫描这个"
    ↓
AI:  "开始扫描 /家庭录像...
      [进度条]
      
      扫描完成！发现 125 个视频，总大小 500GB
      
      检测到这些视频包含人物，建议创建花名册：
      • 发现一个小孩（出现频率最高）
      • 发现两位老人
      • 发现一对年轻夫妇
      
      需要我先学习这些人物吗？"
    ↓
用户: "好，学习"
    ↓
AI:  "请告诉我这个小孩的名字："
    ↓
用户: "小宝"
    ↓
AI:  "好的，已学习'小宝'。
      花名册已建立，包含 5 个人物。
      后续视频会自动识别他们。"
```

### 流程 2：日常使用 - 找视频

```
用户: "找上周拍的视频"
    ↓
AI:  "找到 8 个上周的视频：
      1. 生日会.mp4 (2024-08-15)
      2. 公园玩.mp4 (2024-08-14)
      ...
      
      这些视频中，生日会.mp4 包含小宝和爷爷，
      要播放吗？"
    ↓
用户: "播放生日会"
    ↓
[打开视频播放器，自动跳转到人物出现的时间点]
```

### 流程 3：日常使用 - 去重

```
用户: "找重复视频"
    ↓
AI:  "扫描完成，发现 3 组重复：
      
      第1组：完全重复（可节省 2.5GB）
      - 生日会.mp4 (2.5GB)
      - 生日会(1).mp4 (2.5GB) ← 建议删除
      
      第2组：相似视频（可能是剪辑版）
      - 公园玩.mp4 (45分钟, 3.2GB)
      - 公园玩_剪辑版.mp4 (12分钟, 1.8GB)
      
      要删除建议的文件吗？"
    ↓
用户: "删除第1组的重复，第2组保留"
    ↓
AI:  "已删除 生日会(1).mp4，节省 2.5GB 空间"
```

## 📊 数据展示设计

### 统计仪表盘

| 指标 | 展示方式 | 说明 |
|------|---------|------|
| 视频总数 | 大数字 + 图标 | 直观展示库规模 |
| 人物数量 | 头像墙 | 花名册可视化 |
| 重复视频 | 警告色卡片 | 提醒用户处理 |
| 存储占用 | 进度条 | 磁盘空间管理 |
| 最近活动 | 时间线 | 操作历史 |

### 视频卡片信息

每个视频显示：
- 缩略图（可悬停预览）
- 文件名
- 时长
- 人物标签（头像小图标）
- 重复警告（如有）
- 拍摄日期

## 🎨 设计原则

### 1. 不侵入原则
- **默认只读**：不移动、不删除、不重命名原文件
- **可选操作**：删除/整理需要用户明确确认
- **数据隔离**：所有元数据存在独立的 data/ 目录

### 2. 渐进式智能
- **首次使用**：引导扫描、建立花名册
- **日常使用**：自然语言快速操作
- **高级功能**：专业设置隐藏在设置页

### 3. 透明化AI
- **说人话**：AI解释它在做什么
- **可干预**：用户随时可以说"等等"
- **可追溯**：所有操作有日志记录

### 4. 家庭场景优化
- **人物为中心**：围绕家人组织视频
- **时间线展示**：按日期、事件查看
- **简单易用**：老人小孩都能用

## 📱 多端适配思路

### Web端（主平台）
- 完整功能
- 适合电脑浏览和管理
- 支持拖拽操作

### 手机端（响应式Web）
- 浏览视频库
- 查看人物
- 简单搜索
- 不适合复杂操作

### 大屏端（TV模式）
- 海报墙展示
- 遥控器友好
- 播放控制
- 类似Emby/Jellyfin体验
